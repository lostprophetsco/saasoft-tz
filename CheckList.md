# CheckList - Форма управления учетными записями

## Основные требования

### ✅ Стек технологий
- [x] Vue.js 3 + Composition API
- [x] TypeScript
- [x] Pinia
- [x] UI фреймворк (PrimeVue) - установлен и настроен

### ✅ Структура страницы
- [x] Заголовок формы и кнопка добавления учетной записи (+)
- [x] Панель с информацией и иконкой 
- [x] Таблица со списком учетных записей

### ✅ Таблица со списком учетных записей
- [x] Заголовки таблицы (Метки, Тип записи, Логин, Пароль)
- [x] Ряды таблицы с учетными записями
- [x] Empty state для пустой таблицы

### ✅ Ряд таблицы с учетной записью
- [x] Метка: input text. Необязательное. Максимум 50 символов. В поле вводятся текстовые метки, через знак ;
- [x] Тип записи: select. Обязательное. Значения: LDAP и Локальная
- [x] Логин: input text. Обязательное. Максимум 100 символов.
- [x] Пароль: password. Обязательное. Выводится при Тип записи = Локальная. Максимум 100 символов. Null по дефолту
- [x] Кнопка удаления данной учетной записи.

### ✅ Логика работы
- [ ] При нажатии "+" в конец таблицы добавляется новая пустая запись без кнопки удалить
- [ ] При нажатии кнопки удаления запись полностью удаляется
- [ ] Валидация обязательных полей при потере фокуса
- [ ] Красная обводка для невалидных полей
- [ ] Сохранение/обновление при валидности

### ✅ Обработка данных
- [ ] Поле "Метка" преобразуется в массив объектов `{ text: "элемент метки" }`
- [ ] Разделитель меток - символ `;`
- [ ] Остальные поля сохраняются как текст
- [ ] Сохранение в Pinia store
- [ ] Сохранение в localStorage для персистентности
- [ ] Восстановление данных при перезагрузке страницы

### ✅ Особенности поведения
- [x] Так как при создании учетной записи Тип записи не указан - поле Пароль не отображается и значение null
- [x] При выборе "Локальная" поле "Пароль" показывается и сохраняется
- [x] При выборе "LDAP" поле "Пароль" скрывается и сохраняется как null
- [x] Динамическое отображение/скрытие поля пароля (colspan у логин)
- [x] Кнопка удалить в последней колонке отображается только у сохраненной учетной записи
- [x] Если ни у одной учетной записи нет пароля - скрываем всю колонку, вместе с заголовком таблицы
- [x] Двухфлаговая система: isReadyForSave (готовность к сохранению) / isSaved (сохранено в хранилище)
- [x] Кнопка удаления появляется только после сохранения в Pinia
- [x] Пароль с возможностью просмотра (toggleMask)
